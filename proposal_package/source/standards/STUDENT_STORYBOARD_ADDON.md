# Student Storyboard Add-On (Cartoon Narrative)

Purpose:
Use this short visual story in the management deck to show real parent/student pain points and how the Bluetrace platform solves them.

## Story Format (6 Panels)

1. Morning uncertainty (real problem)
- Visual: Parent at home checking time, bus not yet visible, stress on face.
- Caption: "Did my child board safely?"
- Problem signal: No trusted real-time checkpoint.

2. Smart wearable checkpoint (Bluetrace response)
- Visual: Child taps wearable at bus entry point.
- App event: "Boarded at 7:42 AM" appears in parent app.
- Value: Immediate trust and fewer panic calls.

3. School day updates (academic continuity)
- Visual: Teacher posts homework/circular from teacher app.
- App event: Parent and student receive one consolidated notification.
- Value: No missed WhatsApp messages or fragmented communication.

4. Exception handling (real-time issue)
- Visual: Bus delay / route change alert with acknowledgement button.
- App event: "Route delayed 15 minutes - Acknowledge".
- Value: Faster response and accountable communication.

5. Safe pickup closure (confidence loop)
- Visual: Pickup checkpoint confirms student handover.
- App event: "Picked up at 4:18 PM" in parent app.
- Value: End-to-end safety confirmation.

6. Management outcome view (board-level value)
- Visual: Principal dashboard with trends: on-time routes, alert response time, acknowledgement rates.
- Value: Data-backed governance and stronger school reputation.

## App Screen Mapping (for design team)
- Screen A: Parent home timeline (boarding, alerts, pickup)
- Screen B: Teacher quick publish (homework/circular)
- Screen C: Transport ops panel (route status + exceptions)
- Screen D: Principal KPI dashboard (safety + engagement metrics)

## Messaging Rules
- Keep language safety-first, student-centered, and non-technical.
- Use event-based monitoring wording, not continuous surveillance wording.
- Mention consent workflow and role-based access once in footer.
- Keep each caption under 12 words for slide readability.

## Suggested Deck Placement
- Add after "Role-Based User Experience" slide.
- Follow with "Management Dashboard and Decision Layer" slide.
- Close with pilot scope: one grade + one route + one month baseline.
